# 校园二手书交易平台 v1.0 - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
随着高校学生购书需求的增长，新书价格较高，二手书市场潜力巨大。建立一个专门的校园二手书交易平台，可以帮助学生以更低的价格获取所需教材和课外书籍，同时帮助有闲置书籍的学生变现。

### 1.2 项目目标（v1.0）
为在校学生提供一个**安全、便捷**的二手书交易渠道，实现核心的**发布、浏览、购买**闭环。

### 1.3 目标用户
- 高校在校学生（本科、研究生）
- 即将毕业需要处理闲置书籍的毕业生

## 2. 功能需求（v1.0 核心功能）

### 2.1 用户模块（基础版）

#### 2.1.1 用户注册
- 支持校园邮箱注册（限制学校域名 @gzus.edu.cn）
- 支持手机号注册
- 设置登录密码（6-20位）
- 注册成功后自动登录

#### 2.1.2 用户登录
- 账号密码登录（支持邮箱/手机号）
- 记住密码功能

#### 2.1.3 用户信息管理（简化版）
- 查看个人资料（昵称、头像）
- 修改昵称、头像

### 2.2 书籍模块（v1.0 核心）

#### 2.2.1 书籍发布
| 字段 | 说明 | 必填 |
|------|------|------|
| 书名 | 书籍名称 | 是 |
| 作者 | 书籍作者 | 否 |
| ISBN | 书籍ISBN码 | 否 |
| 分类 | 教材类/考研资料/课外阅读/其他 | 是 |
| 书籍成色 | 全新/九成新/八成新/七成新/及以下 | 是 |
| 价格 | 售价（元） | 是 |
| 书籍描述 | 书籍详情描述 | 否 |
| 交易方式 | 仅自提/仅快递/自提+快递 | 是 |
| 库存数量 | 默认为1 | 是 |
| 书籍图片 | 最多上传3张 | 否 |

#### 2.2.2 书籍浏览
- 首页展示最新发布的书籍列表（分页，每页20条）
- 分类页面按分类展示书籍
- 点击书籍进入详情页

#### 2.2.3 书籍搜索
- 支持按书名模糊搜索
- 支持按分类筛选
- 支持按价格区间筛选
- 支持按书籍成色筛选

#### 2.2.4 书籍详情
- 展示书籍基本信息（书名、作者、分类、成色、价格）
- 展示书籍描述
- 展示书籍图片（最多3张）
- 展示卖家信息（昵称、注册时间）
- **立即购买** 按钮

### 2.3 订单模块（v1.0 简化版）

#### 2.3.1 创建订单
- 立即购买流程：
  1. 选择收货地址（新增收货地址）
  2. 确认订单信息
  3. 提交订单
- 订单状态：待付款 → 待发货 → 待收货 → 已完成

#### 2.3.2 订单管理
- 查看订单列表（全部/待付款/待发货/待收货/已完成）
- 查看订单详情
- **取消订单**（仅待付款状态）
- **确认收货**（待收货状态）
- 订单状态流转：
  ```
  待付款 → 待发货 → 待收货 → 已完成
       ↓
     已取消（用户取消）
  ```

#### 2.3.3 收货地址管理
- 新增收货地址（姓名、电话、地址）
- 查看收货地址列表
- 设置默认收货地址

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间不超过3秒
- 搜索响应时间不超过1秒

### 3.2 安全需求
- 用户密码加密存储（MD5/SHA256）
- 防止SQL注入

### 3.3 界面设计
- 简洁清晰的界面设计
- 移动端响应式布局

---

## 4. 技术架构

### 4.1 前端
- **Vue 3** + Composition API
- **Element Plus** UI组件库
- **Pinia** 状态管理
- **Vue Router** 路由管理

### 4.2 后端
- **Python Flask**
- **Flask-SQLAlchemy** ORM
- **SQLite 3** 数据库

### 4.3 目录结构
```
CampusBook/
├── frontend/          # 前端项目
│   ├── src/
│   │   ├── views/    # 页面组件
│   │   ├── router/   # 路由配置
│   │   ├── api/      # API接口
│   │   ├── stores/   # Pinia状态管理
│   │   └── assets/   # 静态资源
│   └── public/
├── backend/           # 后端项目
│   ├── app/
│   │   ├── models/   # 数据模型
│   │   ├── routes/   # 路由
│   │   └── utils/    # 工具函数
│   └── instance/     # 数据库文件
└── docs/             # 文档
```
